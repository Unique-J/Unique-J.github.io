JavaScript作用域(1)

发表于 2017-04-27   |     |   次阅读

JavaScript 作用域是 JavaScript 中查找变量的一套规则．

词法作用域

词法作用域就是定义在词法阶段的作用域．

示例

  • 全局作用域中只有一个标识符: foo
  • foo 所创建的作用域中有三个标识符: a 、 bar 和 b
  • bar 所创建的作用域中只有一个标识符: c

    查找

    查找变量从当前作用域开始，若没有找到则从上一级作用域继续查找，直到全局作用域．并且如果一个变量在当前作用域以及父级作用域中都存在，则当前作用域中的变量会覆盖父级作用域中的变量．

    欺骗

    JavaScript 中有两种欺骗词法作用域的方法．
  1. eval

eval 可以动态的执行一段代码．示例中通过 eval 在 foo 中对变量 b 进行了声明，此时屏蔽了外部的 b，因此会输出 ‘1,3’ 而非 ‘1,2’.

注： 严格模式下 eval 无法修改所在的作用域

  1. with
    通过 with 可以方便的访问对象的属性：

但是 with 也会有副作用：

o1 中有 a 属性，故可以正常赋值；但 o2 中没有 a 属性，故输出 o2.a 时为 undefined.

注： 若对象属性不存在， with 会将声明的变量添加到 with 所处的函数作用域中

这两个机制的副作用是引擎无法在编译时对作用域查找进行优化，使用这其中任何一个机制都将导致代码运行变慢，因此不要随便使用。
